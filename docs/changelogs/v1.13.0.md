## 🗳️ Sistema Reazioni agli Articoli

Implementato il primo livello di interazione visitatore-contenuto: un sistema di **reazioni a 5 tipi** visibile in fondo ad ogni articolo, tra il contenuto e il box CTA PayPal/Condividi.

---

## Nuovi File

### `public/api/migrate_reactions.php` [NUOVO — eseguire via browser, poi eliminare]
- Script di migrazione DB che crea la tabella `article_reactions`
- Eseguito una tantum via browser dopo il deploy FTP
- Già eseguito e rimosso dal server ✅

### `public/api/reactions.php` [NUOVO]
- Endpoint REST senza autenticazione
- `GET ?article_id=X` → conteggi per ogni tipo di reazione + lista reazioni già date dal visitatore corrente
- `POST { article_id, reaction }` → toggle (aggiunge o rimuove la reazione)
- **Anti-spam:**
  - `UNIQUE KEY (article_id, voter_hash, reaction)` sul DB — impedisce duplicati a livello database
  - `voter_hash = SHA256(IP + User-Agent)` — identificazione anonima GDPR-compliant (nessun dato personale memorizzato)
  - Rate limiting: max 20 azioni per IP nell'ultimo minuto

### `src/components/ReactionBar.tsx` [NUOVO]
- 5 reazioni con **icone SVG inline** (nessuna emoji — scelta obbligata per compatibilità cross-platform Windows)
- Tipi e icone:

| Chiave | Icona SVG | Etichetta | Colore |
|:---|:---|:---|:---|
| `thumb` | Pollice Su | Utile | Sky / Azzurro |
| `heart` | Cuore | Bello | Rose / Rosa |
| `fire` | Fiamma | Interessante | Orange |
| `think` | Punto interrogativo | Fa pensare | Violet |
| `game` | Controller | Game-related | Verde (brand) |

- **Optimistic Update**: il contatore si aggiorna istantaneamente al click senza aspettare la risposta del server; in caso di errore viene eseguito il rollback
- **Tooltip animato** con Framer Motion al hover su ogni bottone
- **Accessibilità**: `aria-label` con etichetta + contatore, `aria-pressed` per lo stato attivo
- **Touch target**: min 44×44px su mobile
- **Persistenza locale**: il componente legge i dati iniziali dal loader (nessuna fetch aggiuntiva al mount)
- **Posizione nel layout**: tra il foglio di lettura e il box CTA PayPal/Condividi

## Schema DB — Nuova Tabella

```sql
CREATE TABLE article_reactions (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    article_id  INT NOT NULL,
    reaction    VARCHAR(20) NOT NULL,  -- 'thumb' | 'heart' | 'fire' | 'think' | 'game'
    voter_hash  VARCHAR(64) NOT NULL,  -- SHA256(IP + User-Agent)
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY unique_vote (article_id, voter_hash, reaction),
    INDEX idx_article (article_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## Note Tecniche

- Le emoji sono state **scartate** in favore di SVG inline per garantire rendering uniforme su Windows (dove le emoji non sono supportate nativamente in molti contesti)
- Il sistema è **stateless lato server**: non richiede sessione, cookie server-side né autenticazione
- Il `voter_hash` è calcolato al volo a ogni richiesta — non è memorizzato separatamente né associabile a un utente
- La build Vite v7.3.1 completata senza errori TypeScript ✅
- Deploy e migrazione DB eseguiti e verificati in produzione ✅
