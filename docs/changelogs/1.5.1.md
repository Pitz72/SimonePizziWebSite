# Changelog v1.5.1 - Patch Sicurezza Forense & Feature Editor

**Data Rilascio:** 04 Marzo 2026

## 🎯 Obiettivo della Release
Questa release (*maintenance update*) interviene immediatamente su vulnerabilità classificate come "Medie" emerse durante l'ispezione forense del codice al backend API avvenuta il 04 Marzo 2026. L'obiettivo primario è state l'incremento di affidabilità e sanitizzazione in fase procedurale del CMS e l'introduzione di una gradita quality-of-life feature all'editor dei contenuti (tool citazione).

## 🛠 Sicurezza & Fix Realizzati

### 1. Fix Spoofing Estensioni (Magic Bytes) - API Upload
- **Problema originario (Criticità Media)**: Lo script protetto `upload.php` si fidava ciecamente dell'estensione terminale del file uploadato via multipart/form-data. Questo esponeva il sistema al rischio teorico di Content-Type spoofing (file eseguibili rinominati in `.jpg`).
- **Soluzione applicata**: È stato adottato il blocco di ispezione **MIME Type interno** avvalendosi della funzione server native `mime_content_type`. Il backend ora fruga dentro i "Magic Bytes" temporanei accertandosi che il contenuto fisico coincida strettamente e lecitamente con le whitelist di estensioni approvate. 

### 2. Fix Typo Dichiarazione Array in Salvataggio Post - API Articles
- **Problema originario (Criticità Media)**: Il controller PHP `articles.php` alla riga 112 presentava un clamoroso banale refuso testuale ($is_featured tentava di castare una stringa booleana attingendo dal parametro `$_data` inesitente, invece di `$data`). L'operatore assumeva fallback permanente a `0`, disabilitando silenziosamente la spunta "Metti in Vetrina" per tutti i NUOVI articoli.
- **Soluzione applicata**: Typo corretto. I comandi POST di `createArticle` tornano ad interpretare regolarmente la flag Vetrina.

### 3. Deprecato Filtering Ridondante (Ottimizzazione Frontend)
- **Problema originario**: Il frontend in `useFetchArticles.ts` in home page scaricava la GET array e processava via JS `data.filter((a: any) => a.status === 'published')`. Un ridondante sovraccarico sul client dato che i check di sicurezza per i non-admin erano già attivi SQL-Side in `articles.php` (che non passa affatto né bozze né preview futuri ad anonimi).
- **Soluzione applicata**: Codice ripulito da filtri Javascript pleonastici, la delega dell'onere al PHP server-side si conferma il Single-Source of truth del portale.

## ✨ Nuove Features Editoriali

### 1. Tool Blockquote (Citazione) nell'Editor Visuale
- Iniettato un nuovo pulsante `Quote` ufficiale (di `lucide-react`) nel pannello di impaginazione Rich Text per il Mini-CMS React. 
- Permette al webmaster, in fase di digitazione articolo in `ArticleEditor.tsx`, di tracciare agilmente i blocchi in stile Blockquote HTML standard (già formattati via tailwind-typography) con la semplice pressione d'un bottone senza dover attingere dalla tendina Dropdown degli Headings.
