# Changelog v1.7.0 (04-04-2026)

## Migrazione Database: SQLite → MySQL

### Motivazione
Il database SQLite era ospitato sul server come file `.sqlite` nella cartella `public/api/.data/`. Questa architettura presentava limiti di scalabilità e non era ottimale per un hosting condiviso standard. La migrazione a MySQL dedicato (host: `mysql.runtimeradio.it`) garantisce maggiore affidabilità, performance e separazione tra codice e dati.

---

### Modifiche Backend

#### `public/api/db.php` — Riscrittura connessione
- Rimossa connessione SQLite (`PDO('sqlite:...')`)
- Rimossi i `PRAGMA` SQLite (`journal_mode`, `foreign_keys`) non applicabili a MySQL
- Connessione ora via `DB_DSN`, `DB_USER`, `DB_PASS` definite in `config.php`
- Aggiunto `PDO::ATTR_EMULATE_PREPARES => false` per maggiore sicurezza delle query preparate

#### `public/api/auth.php` — Compatibilità MySQL
- Fix funzione data rate limiting: `datetime('now', '-15 minutes')` → `DATE_SUB(NOW(), INTERVAL 15 MINUTE)`

#### `public/api/analytics.php` — Compatibilità MySQL
- Fix funzione data trend settimanale: `date('now', '-6 days')` → `DATE_SUB(CURDATE(), INTERVAL 6 DAY)`

### Nuovi File

#### `public/api/config.php` _(gitignored, rimosso da dist)_
Contiene le costanti di connessione al database MySQL (`DB_DSN`, `DB_USER`, `DB_PASS`).
Deve risiedere permanentemente sul server. Non committare mai su Git.

#### `public/api/config.example.php` _(in repo, in dist)_
Template pubblico di `config.php` con valori placeholder. Utile come riferimento per nuove installazioni.

#### `public/api/migrate_to_mysql.php` _(rimosso da dist, da eliminare dopo l'uso)_
Script one-shot di migrazione completa:
- **Fase 1:** Crea tutte le tabelle MySQL con schema aggiornato (`InnoDB`, `utf8mb4`, `AUTO_INCREMENT`, `ON UPDATE CURRENT_TIMESTAMP`)
- **Fase 2:** Legge ogni tabella dal database SQLite esistente e inserisce i dati in MySQL (`INSERT IGNORE` per sicurezza)
- **Fase 3:** Verifica e mostra i conteggi finali per ogni tabella

### Schema MySQL — Novità rispetto a SQLite

| Campo | SQLite | MySQL |
|---|---|---|
| Chiavi primarie | `INTEGER PRIMARY KEY AUTOINCREMENT` | `INT AUTO_INCREMENT PRIMARY KEY` |
| Timestamp default | `TEXT DEFAULT (datetime('now'))` | `DATETIME DEFAULT CURRENT_TIMESTAMP` |
| Auto-update timestamp | non supportato | `updated_at DATETIME ... ON UPDATE CURRENT_TIMESTAMP` |
| Engine | n/a | `InnoDB` (FK enforcement nativo) |
| Charset | n/a | `utf8mb4` (supporto completo Unicode/emoji) |

### Procedura Deploy v1.7.0

1. `npm run build` — genera `dist/` pulita (senza `config.php` e script di migrazione)
2. Carica `dist/` sul server via FTP come al solito
3. Carica **manualmente** via FTP solo questi due file in `/api/`:
   - `public/api/config.php`
   - `public/api/migrate_to_mysql.php`
4. Dal browser: `simonepizzi.it/api/migrate_to_mysql.php`
5. Verificare output "Migrazione completata!"
6. **Eliminare** `migrate_to_mysql.php` dal server
7. **NON eliminare** `config.php` — il sito ne ha bisogno per connettersi
