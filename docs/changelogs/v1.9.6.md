# v1.9.6 — SEO Core Files & The Robots Mystery

**Release Date:** 23 Aprile 2026  
**Focus:** Generazione fisica di robots.txt e sitemap.xml  
**Status:** 🟠 Investigazione in corso (Robots 404 Issues)

---

## Sommario Tecnico
L'obiettivo della v1.9.6 era eliminare la dipendenza dai rewrite dinamici per i file core SEO, generando file fisici tramite lo script di prerendering. 

**Stato attuale:**
- ✅ **sitemap.xml**: Funzionante (fisico o via smart-fallback).
- ❌ **robots.txt**: Fallimento critico. Nonostante le regole di fallback e la generazione fisica, il server continua a dirottare la richiesta verso la SPA (React), risultando in un 404 personalizzato.

---

## Cronologia Investigazione

### 1. Implementazione Generazione Fisica
Abbiamo modificato `api/prerender.php` per catturare l'output di `robots.php` e `sitemap.php` e scriverli nella root del server.
- **Risultato locale:** Successo.
- **Risultato produzione:** Incoerente.

### 2. Smart Fallback .htaccess
Introdotta logica `RewriteCond %{REQUEST_FILENAME} !-f` per servire il PHP se il file statico manca.
- **Comportamento anomalo:** Il server ignora la regola per `robots.txt` ma processa correttamente quella per `sitemap.xml`.

### 3. Verifica Live (Browser Subagent)
L'analisi diretta sul sito `simonepizzi.runtimeradio.it` ha confermato:
- `/robots.txt` -> 404 React (RootBoundary attivata).
- `/sitemap.xml` -> 200 OK (XML visibile).
- `/robots.php` -> 200 OK (Generatore manuale funzionante).

---

## Ipotesi e Prossimi Passi

1. **Caching del Server:** È possibile che il server (DreamHost) stia memorizzando una versione negativa (404) di `robots.txt`.
2. **File Fantasma:** Verificare tramite client FTP se esiste un file `robots.txt` con permessi errati che impedisce sia la lettura che il rewrite.
3. **Priorità Regole:** Provare a spostare le regole di robots sopra ogni altra istruzione, incluso `RewriteBase`.

---

## Note per lo sviluppatore
Il sistema di **ErrorBoundary** (RootBoundary) implementato in questa versione funziona correttamente e intercetta i fallimenti di caricamento, fornendo una UX superiore anche in caso di bug.
