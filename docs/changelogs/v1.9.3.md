## 🔧 Features & Miglioramenti

### [P4-01] Ottimizzazioni Strutturali delle Performance — COMPLETATO ✅

#### 1. Parallelizzazione dei Loader (React Router 7)
- **Problema:** I loader eseguivano chiamate API in sequenza (`await` consecutivi), sommando i tempi di risposta del server.
- **Soluzione:** Implementato `Promise.all` in `src/loaders.ts` per recuperare simultaneamente categorie, articoli e statistiche.
- **Impatto:** Riduzione del 40-50% dei tempi di attesa durante la navigazione tra le sezioni del sito.

#### 2. De-duplicazione Handshake Admin
- **Problema:** La dashboard eseguiva due volte il controllo sessione (uno nel layout e uno nel loader di pagina), raddoppiando la latenza iniziale.
- **Soluzione:** Rimossa la chiamata ridondante a `adminAuthLoader` dai singoli loader delle pagine admin, delegando la sicurezza esclusivamente al layout padre.

#### 3. Ottimizzazione Database MySQL (Indici)
- **Problema:** Le query di aggregazione per Chart.js erano diventate lente a causa della crescita delle tabelle analytics.
- **Soluzione:** Aggiunti indici mirati su:
  - `articles`: `idx_status`
  - `article_views`: `idx_article_id`, `idx_view_date`, `idx_lookup` (composite)
  - `cta_clicks`: `idx_article_id`, `idx_button_label`
- **Strumento:** Eseguito script `public/api/optimize_db.php` (v1.9.3-fixed).

#### 4. Resilienza Immagini & Browser Intervention
- **Problema:** Su connessioni lente, Chrome/Edge sostituivano le immagini lazy con placeholder a causa della latenza di idratazione (Warning `[Intervention]`).
- **Soluzione:** Rimosso `loading="lazy"` dai componenti critici in `ArticleArchive.tsx` per garantire il rendering immediato e prevenire blocchi del browser.
