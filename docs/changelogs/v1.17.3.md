# Changelog v1.17.3 (28 Aprile 2026)

## Fix ricerca admin — input inceppato

### Problema

Nella pagina **Gestione Articoli** del pannello admin, digitare del testo
nella barra di ricerca causava un comportamento anomalo: dopo poche lettere
l'input si bloccava o perdeva il focus, rendendo impossibile completare
la query.

### Causa

L'input era un controlled component il cui `value` era letto direttamente
dai `searchParams` di React Router (`q = searchParams.get('q')`).
Ogni carattere digitato chiamava `setSearchParams`, che in React Router
data-router mode lancia una navigazione completa e ri-esegue il loader
(chiamata API al backend). Durante il ciclo di fetch il componente si
re-renderizzava, causando la perdita del focus e il "blocco" dell'input.

### Fix — `src/pages/admin/ArticlesList.tsx`

Introdotto uno stato locale `searchInput` disaccoppiato dall'URL:

- L'input ora legge/scrive `searchInput` (state React locale) → aggiornamento
  istantaneo senza navigazioni.
- Un `useEffect` con debounce di **400ms** propaga il valore in `searchParams`
  solo dopo che l'utente ha smesso di digitare, attivando il loader una
  sola volta per query.
- Un secondo `useEffect` mantiene `searchInput` sincronizzato quando i
  `searchParams` cambiano dall'esterno (es. reset filtri con il pulsante X).

### Note deploy

Solo aggiornamento frontend. Nessuna modifica PHP o DB.
Caricare la `dist/` aggiornata sul server.
