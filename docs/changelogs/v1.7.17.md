## Changelog Dettagliato

### 🐛 Bug Fix (Incidente Risolto)

- **Risoluzione Errore 500 in Settings**:
  - **Problema**: Identificato un crash del server (Status 500) durante il caricamento delle impostazioni dovuto alla mancanza della tabella `app_settings` nel database di produzione. Il problema era causato dalla rimozione automatica dello script di migrazione durante il processo di build (`clean-dist.js`).
  - **Soluzione**: Implementata l'**Auto-Inizializzazione Resiliente** in `settings.php`. L'API ora è in grado di rilevare la mancanza della tabella al volo, creandola e popolandola con i valori di default senza necessità di interventi manuali sul database.
  - **Risultato**: Sistema ora "Plug-and-Play", stabile sia in ambiente di sviluppo che di produzione.

### 🛠️ Migliorie Tecniche

- **Backend Robustness**:
  - Rafforzati i controlli PDO durante le operazioni di lettura/scrittura delle impostazioni app.
  - Ottimizzazione della logica di `INSERT ... ON DUPLICATE KEY UPDATE` per garantire la coerenza dei dati anche in caso di salvataggi rapidi.
