# Changelog v1.6.2 — Fix Content Security Policy (Google Fonts)

**Data:** 30 Marzo 2026
**Tipo:** Hotfix (Security Headers)
**Versione precedente:** 1.6.1

---

## Panoramica

La v1.6.2 corregge un errore nella Content Security Policy (CSP) di `.htaccess` che bloccava il caricamento del font **Poppins** da Google Fonts, causando avvisi ripetuti in console e un percettibile rallentamento del sito.

---

## 1. Blocco del Font Poppins — Causa Radice

### Sintomi

- Avvisi ripetuti in console del browser su tutte le pagine:

  ```text
  Loading the stylesheet 'https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;600;700&display=swap'
  violates the following Content Security Policy directive: "style-src 'self' 'unsafe-inline'".
  Note that 'style-src-elem' was not explicitly set, so 'style-src' is used as a fallback.
  The action has been blocked.
  ```

- Rallentamento percepibile del sito (il browser ritentava il caricamento del CSS bloccato, generando ricalcoli di layout del font sistema di fallback).

### Causa Tecnica

Il file `index.html` importa il font Poppins tramite un tag `<link>` esterno:

```html
<!-- index.html, riga 10 -->
<link href="https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;600;700&display=swap" rel="stylesheet">
```

  La CSP in `.htaccess` era configurata con:

```text
style-src 'self' 'unsafe-inline'
font-src  'self' data:
```

Questa configurazione consentiva solo CSS serviti dallo stesso dominio (`'self'`) o inline. Qualsiasi foglio di stile esterno — incluso quello di Google Fonts situato su `fonts.googleapis.com` — veniva bloccato dal browser prima ancora di essere scaricato.

Ugualmente, anche se il CSS fosse stato caricato, i file `.woff2` dei font veri e propri sono serviti da un secondo dominio separato (`fonts.gstatic.com`), anch'esso non presente nella direttiva `font-src`, il che avrebbe comunque impedito il rendering del font.

### Trigger

La problematica era già presente nel progetto sin dall'introduzione della CSP (v1.5.10), ma è diventata critica e percepibile solo al caricamento di un nuovo progetto nella sezione "Tutti i Progetti" in data 30/03/2026, momento in cui l'utente ha notato il rallentamento e gli avvisi in console.

---

## 2. Fix Applicato

### File modificato: `public/.htaccess`

**Prima:**

```apache
Header always set Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https: blob:; connect-src 'self'; font-src 'self' data:; object-src 'none'; frame-ancestors 'self'; base-uri 'self'"
```

**Dopo:**

```apache
Header always set Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' data: https://fonts.gstatic.com; img-src 'self' data: https: blob:; connect-src 'self'; object-src 'none'; frame-ancestors 'self'; base-uri 'self'"
```

### Dettaglio delle modifiche CSP

| Direttiva | Prima | Dopo | Motivo |
| :--- | :--- | :--- | :--- |
| `style-src` | `'self' 'unsafe-inline'` | `'self' 'unsafe-inline' https://fonts.googleapis.com` | Permette il download del CSS di Google Fonts (è un `<link rel="stylesheet">`) |
| `font-src` | `'self' data:` | `'self' data: https://fonts.gstatic.com` | Permette il download dei file binari `.woff2` da Google CDN |

### Perché questi due domini e non wildcard `https:`

- Aggiungere `https:` a `style-src` equivarrebbe ad azzerare la protezione CSP contro script/stili iniettati da CDN compromesse.
- `fonts.googleapis.com` è il dominio **specifico e stabile** usato da Google per servire il CSS dei font.
- `fonts.gstatic.com` è il dominio **specifico e stabile** usato da Google per servire i file font binari (`.woff2`, `.ttf`).
- L'approccio allowlist minimale mantiene il livello di sicurezza elevato.

### Impatto sulla sicurezza

La modifica **non indebolisce** la protezione contro attacchi XSS o injection. Le direttive critiche (`script-src 'self'`, `object-src 'none'`, `base-uri 'self'`) rimangono invariate. Si aggiungono esclusivamente i due sottodomini necessari di Google, un provider HTTPS trusted e stabile.
