# 1.4.3 - Hotfix Post-Migrazione (Mapping Dati e API)

**Data:** 1 Marzo 2026

## Panoramica dell'Aggiornamento
A seguito dell'esportazione massiva dei dati storici in formato SQLite, sono stati evidenziati alcuni scollamenti tra il Frontend React e la nuova logica Backend PHP. La versione `1.4.3` risolve definitivamente questi 4 edge-cases funzionali, portando l'applicativo allo stato dell'arte.

## 🐛 Bug Risolti & Ottimizzazioni

### 1. Corpo Articolo Mancante nella Vetrina (Showcase)
**Problema:** Entrando nelle sezioni per categoria (es. Videogiochi), il testo dell'articolo centrale risultava assente, pur essendoci titolo, immagini e tag.
**Causa:** L'endpoint PHP `articles.php` ometteva la colonna `content` nella query della lista, servendo al frontend file HTML/Testuali vuoti.
**Soluzione:** Ottimizzata la query SQL di retrieval includendo esplicitamente il parametro `content` nei risultati.

### 2. Editor Articoli Vuoto in Modalità "Modifica"
**Problema:** Cliccando su "Modifica" di un articolo esistente dal Pannello Admin, i campi Titolo e Corpo rimanevano irrimediabilmente vuoti.
**Causa:** Il componente `ArticleEditor` di React tentava di recuperare il documento tramite un ID numerico (`?id=X`), ma l'endpoint `articles.php` era programmato per ascoltare solo la ricerca tramite slug Stringa (`?slug=X`). 
**Soluzione:** Integrato in PHP il listener combinato per l'ID in modo da soddisfare la chiamata fetch diretta del backend Editor.

### 3. Dashboard Admin con Valori a Zero
**Problema:** La pagina principale del Pannello di Controllo `Dashboard.tsx` mostrava placeholder puntinati `...` e messaggi "In Costruzione" anziché riflettere i veri conteggi degli articoli portati.
**Causa:** Mancanza totale di un bridge API tra la dashboard e i componenti.
**Soluzione:** 
*   Creato da zero l'endpoint `public/api/stats.php` dedicato al logging intelligente del Database.
*   Aggiornato `Dashboard.tsx` con un `useEffect` live che aggiorna istantaneamente i box totalizzatori.
*   Convertiti i vecchi box "In Costruzione" in Shortcut funzionanti verso le rotte `/admin/articles/new` e `/admin/media`.

### 4. Blog Trasformato in "Feed Globale"
**Dinamica:** Originariamente progettato come repository di soli articoli taggati "Blog", in base al feedback è emerso che questa pagina doveva agire come collettore universale per tutto il sito.
**Soluzione:** Svincolata la categoria `BLOG_E_RIFLESSIONI` dal meccanismo di routing in `App.tsx` e `PortfolioShowcase.tsx`. Omettendo il passaggio della categoria selettiva, il componente ora inietta l'intero array di tutti gli articoli in ordine cronologico.
