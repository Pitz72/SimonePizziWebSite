# 1.4.2 - Deploy Sicuro (Prevenzione Sovrascrittura DB)

**Data:** 1 Marzo 2026

## Panoramica dell'Aggiornamento
L'aggiornamento 1.4.2 introduce uno script critico di sicurezza per il processo di rilascio dell'applicazione in produzione. 

## Il Problema (File Copy di Vite)
Vite, durante il processo di build (`npm run build`), copia per definizione tutto il contenuto della cartella `/public` all'interno della cartella di rilascio finale `/dist`.
Questo causava un problema architetturale fatale: il database SQLite in uso (`public/api/.data/database.sqlite`) e i file multimediali degli articoli (`public/uploads`) venivano fisicamente traslati all'interno della cartella di esportazione. 
Se il Webmaster avesse fatto l'upload FTP dell'intera cartella `/dist` aggiornata, avrebbe involontariamente sovrascritto il database Live di Produzione (con i suoi dati e account) rimpiazzandolo con la copia statica locale.

## Soluzione Adottata (Post-Build Hook)
Si è ovviato al problema creando un hook di "Post-Compilazione".
1. Creato lo script Node estraneo al frontend **`clean-dist.js`** posizionato nella root. 
2. Aggiunto l'evocazione `"postbuild": "node clean-dist.js"` nel file `package.json`.

Alla fine di ogni processo di `npm run build`, prima di restituire il controllo al Webmaster, lo script si attiva forzatamente e **elimina fisicamente le directory `.data` e `uploads` dall'interno della cartella `dist/`**, garantendo in termini assoluti che il pacchetto di esportazione non contenga mai il database. 

L'upload FTP è ora un'operazione blindata e priva di rischi.
