# Changelog v1.5.2 - Sblocco Anteprima Bozze (Admin-Only)

**Data Rilascio:** 04 Marzo 2026

## 🎯 Obiettivo della Release
L'obiettivo di questo micro-aggiornamento è migliorare l'esperienza utente ("Quality of Life") del pannello di amministrazione. Finora gli articoli venivano completamente sigillati dalle API se il loro status era `draft` o la data di pubblicazione era nel futuro, rendendo impossibile anche per l'Admin vedere una live-preview della propria impaginazione. Questa patch risolve l'impedimento mettendo in sicurezza la pipeline.

## ✨ Dettaglio Modifiche

### 1. Bypass Permessi in Coda (API Articles)
- **Modifica a `articles.php`**: Alterata la rotta `GET ?slug=...`. Da oggi incrocia la sessione nativa PHP `$_SESSION['user_id']`. 
- **Logica**: Se a richiedere l'articolo tramite link diretto (quindi passando il layer JSON verso React `SingleArticle`) è un visitatore anonimo e l'articolo NON è pubblico (Bozza/Programmato Futuro), il server risponde con `404 - Articolo non trovato (Bozza/Privato)`. Se a inviare la request è il browser dell'amministratore (con cookie sessione valido), la barriera crolla e viene iniettata l'anteprima 1:1.
- *Nota: Nessuna modifica al pattern DB richiesta.*

### 2. Sblocco UI Dashboard
- **Gestore Link in `ArticlesList.tsx`**: Rimosso lo sbarramento visivo lato client (`{article.status === 'published' && ... }`). L'icona `ExternalLink` ("Vedi sul sito") è ora persistentemente accesa e cliccabile di fianco ad ogni entità tabella, anche se in grassetto rosso scuro con su su scritto "Bozza".
