# Changelog v1.4.8 - Hotfix CTA Articles e Loading Ottimizzato

**Data Rilascio:** 01 Marzo 2026

## 🎯 Obiettivo della Release
Questa patch release mira a risolvere un problema di invisibilità dei bottoni di "Call To Action" (Azione Primaria e Secondaria) configurati per ogni articolo e a ottimizzare contemporaneamente le prestazioni di caricamento delle pagine interne tagliando uno spreco architetturale.

## 🛠 Fix Realizzati

### 1. Risolto Bug di Visualizzazione Bottoni ("CTA") negli Articoli
- **Il problema:** I bottoni (verde e grigio) definiti in `ArticleEditor.tsx` non venivano renderizzati all'interno del reader di `SingleArticle.tsx`, sebbene i dati salvati su SQLite risultassero intatti e corretti. Il frontend, scaricando una cache massiva degli articoli (sfruttando l'endpoint di base del backend privo della clausola dei bottoni a fini di leggerezza JSON), auto-censurava l'interfaccia.
- **La Soluzione:**
  - Sviluppata la chiamata `api.getArticleBySlug(slug: string)` all'interno del modulo `api.ts`.
  - Rifattorizzato profondamente il componente `SingleArticle.tsx`. In base alla lettura dell'URL (projectSlug), esso interroga ora attivamente e puntualmente il backend usando `?slug=il-mio-articolo`. In questo modo beneficia di una stringa JSON "100% full-weight" comprensiva dei nodi `button_a_label` e `button_a_link`, e renderizza a schermo l'interfaccia dinamicamente senza appoggiarsi ai filtri massivi in Array `data.find()`.

- Disaccoppiando il fetch dell'articolo da `api.getArticles()` intero, e utilizzando il parametro `slug`, il sito non spreca più memoria RAM (lato client) e computazioni (lato server SQLite) interrogando indiscriminatamente dozzine di articoli per stamparne a video solo uno.  

### 3. Riparazione Errore 400 (Salvataggio Articolo)
- **Il problema:** Durante il salvataggio o l'aggiornamento di un articolo tramite l'Editor in area riservata, il sistema restituiva "ID Mancante" a causa di uno script PHP che si aspettava l'ID esclusivamente tramite protocollo GET nell'URL, ignorando il payload del corpo JSON e causando un blocco operativo (Bad Request).
- **La Soluzione:** Aggiornati i file router `articles.php` nei metodi PUT e DELETE per supportare l'estrazione mista dell'ID e il mapping di sicurezza tramite l'uso del Null Coalescing (`??`), impedendo perdite accidentali di campi opzionali.

### 4. Overhaul Visuale Bottoni Editor
- La sezione dell'Editor riservata all'impostazione dei bottoni CTA mancava di accessibilità. Riprogettata l'intera UI React rimpiazzando i moduli spogli con Layout strutturati, sfondi solidi (verde scuro e nero profondo) ed etichette descrittive per suggerire immediatamente l'area di appartenenza al redattore.
