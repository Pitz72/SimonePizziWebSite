# 1.4.1 - Porting Eredità e Dati Storici su SQLite

**Data:** 1 Marzo 2026

## Panoramica dell'Aggiornamento
La versione 1.4.1 nasce come complemento essenziale al rilascio del Golden Master (v1.4.0). L'obiettivo primario era assicurare che l'impressionante portafoglio di contenuti hardcoded nel file TypeScript (`portfolioData.ts`) non andasse perso, ma venisse travasato integralmente nel nuovo ecosistema **Mini-CMS**.

## Interventi Tecnici 

### 1. Script di Porting Usa-e-Getta (`porting_data.php`)
È stato creato uno script speciale all'interno della cartella protetta `/api/` incaricato di traghettare le informazioni.
* **Mappatura Hardcoded > Database**: I vecchi oggetti JSON annidati (`title`, `description` HTML, `summary`, `imageUrl`) sono stati formattati in array PHP pre-compilati.
* **Auto-Slug Generation**: Lo script genera autonomamente lo slug univoco (URL-friendly) basandosi sui titoli estratti dal file TS originale, assicurando coerenza e funzionamento immediato col motore SEO `index.php`.
* **Traduzione Enum**: Le vecchie categorie basate su Macro React (es. `Category.VIDEOGIOCHI`) sono state convertite in stringhe flat (`videogiochi`) supportate dal campo database e richiamate poi correttamente dai Dropdown frontend.

### 2. Conservazione Layout Speciali Frontend
* Tutta l'impaginazione che un tempo risiedeva sotto la proprietà `description` (spesso codice HTML multiriga iniettato a forza) è colata direttamente nel campo testuale `content` di SQLite. Questo HTML super-raw risponde in modo formidabile da dentro il Database e il nuovo Editor Modificato React saprà estrarre tranquillamente i marcatori in caso di future modifiche amministrative.
* Preservati i tag ausiliari come i Bottoni Personalizzati o i Link Esterni Speciali (Es: *"Materiale Extra"* o collegamenti ad `.epub`), sfruttando i 4 campi dedicati alle due call-to-action del Database (`button_a_label` etc).

## Note Strategiche 
La procedura garantisce continuità. Il file `porting_data.php` ha valenza "one-shot" d'esecuzione singola su browser (`https://mioUrl/api/porting_data.php`). La sua natura insicura prevede e richiede la cancellazione diretta tramite filesystem/FTP subito dopo il feedback di stampa positivo per annullare ogni tipo di rischio su scritture abusive future.
