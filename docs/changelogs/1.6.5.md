# Changelog v1.6.5 (03-04-2026)

### Feature 1: Gestore Categorie Dinamiche

#### Obiettivo
Le categorie (Videogiochi, Software, ecc.) erano hard-coded nell'Enum TypeScript. Ora sono gestite dinamicamente dal DB SQLite e configurabili dall'admin senza toccare il codice.

#### Implementazione Backend
*   **Nuova tabella DB `categories`:** Campi `id`, `name`, `slug`, `sort_order`, `created_at`. Popolata con le 5 categorie originali via script di migrazione.
*   **Nuova API `categories.php`:** Endpoint REST completo — `GET` pubblico (lista ordinata), `POST`/`PUT`/`DELETE` protetti da `Auth::check()`.
*   **Script `migrate_v165.php`:** Crea le tabelle `categories`, `article_views`, `cta_clicks`. Da eseguire sul server al primo deploy, poi eliminare.

#### Implementazione Frontend
*   **Nuovo hook `useCategories.ts`:** Carica le categorie dall'API con fallback ai valori hardcoded (garantisce navigazione immediata anche prima che l'API risponda).
*   **`App.tsx` — Routing Dinamico:** Le route pubbliche `/:categorySlug` e `/:categorySlug/:projectSlug` sono ora generate a runtime dal hook `useCategories`, sostituendo le route statiche basate sull'Enum.
*   **`Header.tsx` — Nav Dinamica:** I link di navigazione sono generati dalla stessa lista di categorie, mantenendo automatica la sincronizzazione con le modifiche admin.
*   **Nuovo componente admin `CategoryManager.tsx`:** UI completa per creare, rinominare, eliminare e riordinare le categorie con pulsanti su/giu.
*   **`AdminLayout.tsx`:** Aggiunta voce "Categorie" nel menu laterale (icona Tag).
*   **`types.ts`:** Aggiunta interfaccia `CategoryItem` per il tipo restituito dall'API.
*   **`api.ts`:** Aggiunti metodi `getCategories()`, `createCategory()`, `updateCategory()`, `deleteCategory()`.

### Feature 2: Dashboard Analitica Avanzata

#### Obiettivo
Arricchire la dashboard admin con metriche di engagement: visualizzazioni per articolo e statistiche di click sui pulsanti CTA.

#### Implementazione Backend
*   **Nuova tabella DB `article_views`:** Campi `article_id`, `ip_hash`, `view_date`. Deduplicazione: stesso IP (SHA-256) + stesso articolo + stessa data non genera visualizzazioni multiple.
*   **Nuova tabella DB `cta_clicks`:** Campi `article_id`, `button_label`, `clicked_at`. Registra ogni click con l'etichetta del pulsante.
*   **Nuova API `analytics.php`:** `POST` pubblico per il tracking (view/click), `GET` admin per le statistiche aggregate (top articoli, trend settimanale, click per bottone).
*   **`stats.php` aggiornato:** Espone `total_views` e `total_clicks` nei dati di sistema (con fallback a 0 se la migrazione non e ancora stata eseguita).

#### Implementazione Frontend
*   **`SingleArticle.tsx`:** Al caricamento di ogni articolo viene inviato un tracking view (fire-and-forget, silenzioso). Al click su ogni pulsante CTA viene inviato un tracking click con l'etichetta del pulsante.
*   **`Dashboard.tsx` — Sezione Analytics:**
    *   Nuove stat card: Visualizzazioni Totali, Click CTA (aggiornati live).
    *   **Top 5 articoli** per numero di visualizzazioni, con contatore.
    *   **Grafico a barre** delle visualizzazioni negli ultimi 7 giorni.
    *   **Breakdown click** per etichetta di pulsante CTA.
*   **`api.ts`:** Aggiunti metodi `trackView()`, `trackClick()`, `getAnalytics()`.
