# Changelog v1.17.1 (27 Aprile 2026)

## Hotfix post-deploy V3 "Terminal Botanico"

Quattro bug riscontrati immediatamente dopo il deploy di v1.17.0.

---

### Bug risolti

#### Cursore invisibile nell'admin dashboard
- `CustomCursor` ora aggiunge la classe `sp-cursor` a `document.body` al mount e la rimuove all'unmount
- Il CSS `cursor: none` è già scopato a `body.sp-cursor` (fix v1.17.0)
- L'admin panel, che non monta `CustomCursor`, mantiene il cursore nativo di sistema

#### Dropdown sottocategorie assente nell'header
- Il componente `Header` era stato riscritto per V3 senza includere la tendina per le sottocategorie
- Nuovo componente `DesktopNavItem`: gestisce sia link semplici (categorie senza figli) sia dropdown (categorie con `subcategories`)
- Dropdown V3: glassmorphism `rgba(5,8,10,0.95)` + blur 20px + border verde tinted
- Include link "Tutti — NomeCategoria" + separatore + link per ogni sottocategoria
- Chiusura automatica al click fuori (mousedown listener globale)

#### Immagini/copertine scomparse in alcune card
- In V3 le card usano l'immagine come sfondo full-bleed con `absolute inset-0`
- Gli articoli senza `imageUrl` (null o stringa vuota) rendevano la card completamente nera
- Aggiunto guard `imageUrl ? <img> : <div fallback gradient>` in tutti i punti:
  - `FeaturedCard` → `HeroCard` (corretto anche da `w-full h-full` a `absolute inset-0`)
  - `FeaturedCard` → `GridCard` articoli
  - `FeaturedCard` → `GridCard` progetti
  - `ArticleArchive` → hero card
  - `ArticleArchive` → grid card

#### Media gallery — errore 401 (non risolvibile via codice)
- Causa: patch sicurezza v1.15.1 ha introdotto `session_version`; sessioni admin precedenti non hanno questa chiave → mismatch → 401
- **Soluzione operativa**: uscire dall'admin panel e rientrare; la nuova sessione contiene `session_version` corretta
