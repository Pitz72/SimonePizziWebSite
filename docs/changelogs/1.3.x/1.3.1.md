# 1.3.1 - Rete API Principale (PHP Controllers)

**Data:** 28 Febbraio 2026

Secondo step della Roadmap di Costruzione del Mini-CMS. In questo passaggio è stata stesa tutta l'infrastruttura d'interrogazione (Il Livello API HTTP) in PHP per fare in modo che il futuro cruscotto React e il sito pubblico possano parlare con il neo-nato database SQLite.

## ⚙️ Dettagli dell'Intervento (Controller API)
1. **`articles.php`**: È stato progettato il controller Master che si occupa delle classiche operazioni CRUD (Create, Read, Update, Delete). 
    *   Le richieste `GET` permetteranno al sito React di prelevare tutti gli articoli o solo quelli appartenenti a una specifica *categoria* passata in Query String. Se l'utente non sarà autenticato, verranno esclusi gli articoli con stato `draft` (bozza).
    *   Gli articoli non avranno conflitti URL: una helper PHP interna genererà fisicamente un URL-Slug univoco (pulendo il Titolo dai caratteri speciali e appendendo un timestamp in caso di collisione).
2. **`upload.php`**: Lo script che gestisce la ricezione dei bit dalla "Media Gallery" dell'Admin. In ricezione, verifica che il file rispetti i limiti di sicurezza Mime-Type (inclusi `PDF`, `ZIP`, `RAR` e `MP3` oltre alle pure immagini) e ne assegna un ID casuale su Disco all'interno di `public/uploads/` prima di censirne la presenza nella tabella database SQLite `media`.
3. **`media.php`**: Lo script di servizio che consentirà al Pannello Admin di sfogliare (`GET`) l'intero catalogo dei file archiviati e di cancellarli fisicamente (`DELETE`), eliminando di concerto i relativi record SQLite associati e il binario reale dentro `/uploads`.
4. **`auth_helper.php`**: Piccola utility Session-based invocata in vetta agli script POST, PUT e DELETE, che fermerà nativamente (Errore `401 Unauthorized`) qualsiasi tentativo di scrittura o cancellazione articoli/media non autorizzato da un token di sessione valido, chiudendo le porte di servizio esposte a web.

## ✨ Stato Attuale
Anche questa patch è "Layer-Only". Prepara il terreno ma non muta l'interfaccia React e non causa rotture operative sul codice in essere. Al culmine dello Step 3, questo livello inizierà ad essere martellato dalla nuova interfaccia di Login e Admin Dashboard del sito. Aggiunta cartella `public/uploads/.gitkeep` solo per referenziarne la traccia su GitHub pur nascondendo i veri e propri file user-generated. Build 1.3.1 testata con successo.
