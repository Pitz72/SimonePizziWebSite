# 1.3.0 - Foundation Backend Mini-CMS

**Data:** 28 Febbraio 2026

Questo è il primo step (fase Inizializzazione Rete) del percorso architetturale per dotare il progetto di un backend vero e proprio, trasformando il sito da una SPA statica a un sistema ibrido dinamico provvisto di un Mini-CMS.

## ⚙️ Dettagli dell'Intervento (Setup Database)
1. **Creazione Layer API PHP**: È stata creata la cartella `public/api/` che ospiterà tutti i microservizi backend in linguaggio PHP. Essendo all'interno della cartella `public`, Vite la conserverà intatta durante le build servendola direttamente ai router finali.
2. **Connettore SQLite (`db.php`)**: È stata istanziata la classe statica `Database` che genererà ed interrogherà fisicamente il file `database.sqlite` sotto `public/api/.data/`. Lo script genera automaticamente un `.htaccess` per proteggere il file DB dai download diretti non autorizzati e, sulla base delle lezioni apprese in "Disintelligenza" e "SitoRuntime", utilizza la modalità `DELETE` journal per aggirare i problemi di file lock degli hosting condivisi classici.
3. **Database Schema (`init_db.php`)**: È stato steso il Blueprint dell'intero sistema. Lo script API è pronto ad auto-costruire le tabelle necessarie in SQLite:
    *   `users`: Per login admin protetto da Password Hash (account di base `admin` / `admin2026`).
    *   `articles`: Il fulcro del CMS. Contiene campi per titolo, slug, testi in RTF, copertine, categorie, doppi-bottoni e la logica "vetrina" (`is_featured`).
    *   `media`: Archivio tracce dei file multi-formato (Immagini, PDF, ZIP).
    *   `messages` e `subscribers`: Per i moduli di contatto e profilazione newsletter.

## ✨ Stato Attuale
Il frontend è per ora immutato, ma l'impalcatura del backend è stata gettata e può già essere interrogata da un server locale (es. MAMP puntato sulla cartella root del progetto su porta 8888 o simili).
La build 1.3.0 non presenta incompatibilità coi flussi React legacy della 1.2.x in attesa dei prossimi step dove agganceremo i data source alle UI.
