# 1.3.2 - Auth & Foundation UI (React Admin Panel)

**Data:** 28 Febbraio 2026

Questo step introduce prepotentemente il Frontend nella vita del Mini-CMS, creando finalmente una "stanza di controllo" visiva ed interattiva che dialoga con le API in PHP precedentemente confezionate.

## ⚙️ Dettagli dell'Intervento (Pannello Amministrazione)
1. **API Client (`src/api.ts`)**: Creato l'oggetto `fetchConfig` standardizzato che si occuperà di comunicare col Backend gestendo le policy CORS, i payload JSON e l'header di Credential Handling per mantenere vivi i Cookie di sessione di PHP al momento dell'autenticazione.
2. **Pagine Amministrative**:
    *   `Login.tsx`: Creata una splendida UI "Dark Glassmorphism" per accedere all'area riservata (invoca `api.login()`).
    *   `AdminLayout.tsx`: Costrutto-scudo. Si assicura che ad ogni caricamento l'utente sia autenticato (tramite `api.checkSession()`). Renderizza una Sidebar laterale espandibile e permanente e inietta al centro i sottomoduli.
    *   `Dashboard.tsx`: Una bacheca riassuntiva.
    *   `Settings.tsx`: Modulo cruciale per la sicurezza. Manda un monito all'utente esortandolo a cambiare la password predefinita (`admin2026`) scambiando via API col controller protetto in `settings.php`.
3. **App Router (`App.tsx`)**: Riformattazione architetturale dell'app originaria. Tutte le pre-esistenti view pubbliche sono state incapsulate in uno `<PublicLayout />`, mentre i nuovi percorsi `/admin/*` vivono indipendentemente privi di `Header` e `Footer` classici, sfruttando la nuova plancia di comando.

## ✨ Stato Attuale
Il sito pubblico è immutato (ma incapsulato nel router diversamente). Digitanto `# /admin/login` l'utente atterrerà sulla schermata di login e potrà loggare col superuser "admin" / "admin2026". Lo switch password è pienamente operativo e memorizza con successo in SQLite l'hash di rimpiazzo BCRYPT generato in PHP. Build 1.3.2 passata e testata.
Prossimo passaggio: l'integrazione del VERO editor visuale degli articoli.
