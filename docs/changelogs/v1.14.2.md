# Changelog v1.14.2 (26 Aprile 2026)

## 📁 Riorganizzazione Media & Ottimizzazione Esportazione

Questa versione introduce una ristrutturazione profonda della gestione dei file multimediali e un miglioramento significativo della User Experience nell'area di amministrazione, risolvendo criticità legate alla latenza di compressione del server.

### 🚀 Novità e Miglioramenti

- **Struttura Media Organizzata:** La cartella `/uploads/` è stata suddivisa in sottodirectory logiche per una gestione più pulita e professionale:
  - `/uploads/immagini/`: Destinata a tutti i formati grafici (JPG, PNG, WebP, GIF).
  - `/uploads/documenti/`: Destinata ai file PDF.
  - `/uploads/file/`: Destinata ad archivi (ZIP, RAR) e altri formati (MP3).
- **Logica di Upload Intelligente:** Aggiornato `upload.php` per smistare automaticamente i nuovi caricamenti nelle cartelle di competenza in base al tipo MIME reale del file.
- **Full Content Exporter 2.0:** Allineato il sistema di backup alla nuova struttura. L'esportazione ZIP ora preleva i contenuti direttamente dalle nuove sottocartelle, garantendo pacchetti organizzati.
- **Fix "Pagina Nera" (UX):** Risolto il problema del browser che mostrava una pagina nera/vuota durante la generazione di ZIP pesanti.
  - Implementato overlay di caricamento dinamico con messaggio informativo ("Generazione Pacchetto").
  - Transizione del download a sistema basato su Blob/Fetch, mantenendo l'utente all'interno della dashboard.
- **Raffinamento Interfaccia Admin:** Aggiornate le etichette descrittive nella sezione Impostazioni per riflettere i nuovi percorsi di sistema.

### 🛠️ Interventi Tecnici

- **Script di Migrazione Automatica:** Creato ed eseguito uno script per lo spostamento fisico di 65 file esistenti e l'aggiornamento massivo dei percorsi nel database MySQL.
- **Refactoring API:** Aggiornato `src/api.ts` per gestire il download asincrono dei flussi binari (Blob) e fornire feedback di errore granulare in caso di timeout del server.

### ⚠️ Note Notevoli

- **Browser Intervention:** Identificato un warning `[Intervention]` nella Media Gallery relativo al lazy-loading delle immagini. Il comportamento è gestito nativamente dai browser moderni (Chromium/Edge) per ottimizzare la memoria; non impatta la funzionalità ma posticipa il caricamento degli asset fuori viewport.
