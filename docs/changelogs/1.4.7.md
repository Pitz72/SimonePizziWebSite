# 1.4.7 - Security & Codebase Cleanup (The Trust Protocol)

**Data:** 1 Marzo 2026

## Panoramica dell'Aggiornamento
A seguito di una scrupolosa fase di ispezione autonoma (Agentic Code Audit) e successiva validazione, questa release si concentra esclusivamente sulla sicurezza dell'infrastruttura, sulla prevenzione da abusi malevoli del database e sull'estinzione definitiva del debito tecnico accumulato nelle precedenti fasi di Redesign Editoriale.

## 🛡️ Sicurezza e Resilienza Backend
Il "Mini-CMS" scritto in PHP (cartella `public/api/`) è stato blindato contro i più comuni vettori d'attacco:
1. **Prevenzione Brute-Force (Rate Limiting)**: Implementato un sistema nativo all'interno del controller `auth.php` basato su una nuova tabella passiva SQLite (`login_attempts`). Il sistema ora intercetta l'IP del client e, se rileva 5 tentativi di login errati consecutivi nell'arco di 15 minuti, blocca le richieste silenziando la UI con uno status `429 Too Many Requests`. Questo tutela irreversibilmente l'unico account Admin da script bot-dizionari, senza minimamente intaccare o invalidare l'hash originale della password.
2. **Rimozione Script Usa e Getta**: Eliminato `porting_data.php`. In precedenza lasciato erroneamente aperto al pubblico, rappresentava una potenziale (grave) via d'accesso per eseguire massicci statement di `INSERT` ricorsivi e saturare lo spazio su disco / rallentare l'I/O SQLite.

## 🧼 Root Pollution e Debito Tecnico
La cartella react dei componenti visuali (`src/components/`) è stata purgata chirurgicamente:
- Rimossi fisicamente `PortfolioShowcase.tsx`.
- Rimossi fisicamente `ProjectList.tsx`.
- Rimossi fisicamente `ProjectDetail.tsx`.
*Nota: Sebbene precedentemente scollegati dal router in v1.4.5, la loro persistenza generava "rumore" visivo nell'albero e minacciava future regressioni AI. Il disaccoppiamento legacy si chiude qui.*

## ⚙️ SSR SEO Engine Tuning (`index.php`)
Sviluppato un miglioramento algoritmico "Fail-Safe" per l'interprete PHP che inietta le Twitter Card per i bot Facebook/LinkedIn:
- Disinnescato l'hardcoding forzato dell'URL (prima impuntato a `https://simonepizzi.it`) sostituendolo con un rilevatore dinamico del Protocollo (`$_SERVER['HTTPS']` vs port `443` vs `http://`).
- Sostituita la generica `cover-default.jpg` (non esistente in public) con l'effettivo `Simone-Pizzi.png` come copertina paracadute in caso l'articolo interrogato ne sia sprovvisto.
- Implementata la normalizzazione algebrica degli Slash `prefix` per evitare che, inserendo le Cover URL in DB durante la stesura, l'innesco `http_host` in PHP generasse path rotti.
