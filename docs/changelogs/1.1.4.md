# 1.1.4 - System Architecture Refactoring & Data Cleanup

**Data:** 28 Febbraio 2026

Sebbene questa release non introduca nuove feature visibili all'utente finale (UI/UX inalterata), rappresenta uno degli interventi curativi (DevOps/Architetturali) più importanti dall'inizio del filone 1.1.x.

## 🧹 A. Deep Cleaning (Rimozione Zavorre e Cache)
Il repository accumulava diversi megabyte di file spazzatura, vecchi snapshot e asset pesanti non usati dal frontend di produzione.
Sono stati rimossi permanentemente tramite scripting di basso livello i seguenti elementi "orfani":
*   **[DELETE] Cartella `/backup`**: Conteneva versioni HTML obsolete del sito e cloni di log vecchi. Eliminata per scaricare la root.
*   **[DELETE] Cartella `/import`**: Parcheggio per file grezzi non ottimizzati e PSD (immagini fino a 3 MB l'una). Eliminato in blocco per snellire il Git index.
*   **[DELETE] Cartella `/temp`**: Cartellina provvisoria rimasta inutilmente a macerare in root. Spazzata via.
*   **[DELETE] Il file `rlm.png` "clone"**: Rimosso dalla Root un clone grezzo da 5 MegaByte della copertina di *Runtime Live Machine*, lasciando in vita solo la variante ottimizzata puntata in `public/images/`.

## 📂 B. Riorganizzazione della Root Directory
Sono stati mitigati gravosi problemi di sdoppiamento cartelle. Cartelle cruciali di React stavano in root generando entropia.
*   **[DELETE] Cloni scompattati**: Rimosse le cartelle `/components`, `/data`, `/utils` e il file `types.ts` scompattati malamente in root. Il compilatore ora si appoggia unicamente sulle direttrici sicure all'interno dello scope nativo `/src/`.
*   **[NEW] Utilities Hub**: Creata la cartella operativa `/scripts/` dove riporre tool a sé stanti (es. `optimize_image.ps1` è stato spostato in `scripts/optimize_image.ps1`).

## 📚 C. Documental Hub Revolution (Single Source of Documentation)
La documentazione è stata gerarchicamente ricostruita secondo principi di archiviazione accademica e informatica:
*   **[NEW] `docs/README.md` (Master Index)**: Creato l'Indice Master. La cartella `/docs` non è più un contenitore informe ma uno schedario gerarchico. Questo index censisce tutta la timeline (fino al log 1.0.1) e instrada sviluppatori e Bot.
*   **[NEW] `docs/structure_index.md` (Structure Mapping)**: Scritto manifest strutturale rivolto in particolare all'Intelligenza Artificiale (AI-Guidelines e Agenti Autonomi) per arginare per sempre la *root pollution*, spiegando i confini netti tra la cartella `/src`, `/public` e `/docs`.
*   **[MODIFY] Aggiornamento globale referenze**: I file `README.md` radice e `docs/project_status.md` sono stati profondamente riscritti allacciandosi al neonato ecosistema e allineandosi alle modifiche *Live Machine* precedentemente censite nel meta-dato.

## Note conclusive per Build
Eseguita pipeline di test build via `npm run build` su stack Vite+TS: esito *Success*. Lo scaling up del Data-Driven system risulta compatto, sanificato e documentato al 100%.
