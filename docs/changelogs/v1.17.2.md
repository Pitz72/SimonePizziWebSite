# Changelog v1.17.2 (27 Aprile 2026)

## Fix immagini uploads + nuovo ritratto Hero

### Migrazione path immagini DB

Dopo il deploy di v1.17.0 (V3 "Terminal Botanico") è emerso che le immagini
di cover e corpo articolo erano invisibili. Causa: la cartella upload era stata
riorganizzata aggiungendo la sottodirectory `immagini/`, ma i record nel DB
conservavano i vecchi path senza di essa.

Eseguita migrazione in due fasi tramite script PHP temporanei:

**Fase 1 — cover_image** (`fix_uploads_subfolder.php`)
- Tabella `articles.cover_image`: 25 record aggiornati
- Tabella `projects.cover_image`: 10 record aggiornati
- Conversione: `/uploads/file.webp` → `/uploads/immagini/file.webp`
- Gestiti anche path assoluti con dominio (`https://simonepizzi.runtimeradio.it/uploads/...`)
- Record già corretti (art.39) e path statici `/images/` non toccati
- 0 file mancanti su disco

**Fase 2 — body content** (`fix_content_image_paths.php`)
- Tabella `articles.content`: 9 articoli aggiornati, 26 occorrenze totali
- Regex con lookahead negativo per non toccare path già corretti
- Filtro su estensioni immagine (webp, jpg, jpeg, png, gif, avif, svg)
- PDF e altri documenti esclusi dalla sostituzione
- PDF nei link articolo rimangono invariati (serviti via `download.php?id=`)

Entrambi gli script eseguiti con dry-run preventivo (0 errori), poi applicati,
poi cancellati dal server.

### Nuovo ritratto Hero

- **File:** `public/Simone-Pizzi.webp` (sostituzione in-place)
- **Sorgente:** `SimonePizzi.png` (1254×1254px) convertito in WebP 90% via Sharp → 140KB
- **Formato:** quadrato, lighting verde atmosferico — coerente con la palette V3
- **CSS aggiornato in `Hero.tsx`:**
  - `object-bottom` → `object-center` (immagine quadrata, soggetto centrato)
  - Mask ellittica ricalibrata: `radial-gradient(ellipse 72% 78% at 56% 50%, ...)` —
    sfumatura spostata leggermente a destra per lasciare spazio alla fade sinistra
  - `brightness` portato a 0.95 (meno scuro, il lighting dell'immagine è già cupo)
