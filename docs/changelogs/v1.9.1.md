## 🚀 Miglioramenti Performance

### [P2-05] Ottimizzazione Bundle & Code Splitting — COMPLETATO ✅

Per migliorare l'esperienza utente "Premium" e ridurre il peso del primo caricamento (First Contentful Paint), abbiamo implementato una strategia di **Code Splitting** aggressiva.

#### Cambiamenti Architetturali:
1.  **Lazy Loading delle Rotte:** Tutte le pagine dell'area **Admin** e le sottopagine pubbliche (Contatti, Progetti, Articoli singoli) vengono ora caricate solo quando effettivamente richieste dall'utente.
2.  **Riduzione Bundle Iniziale:** Spostando moduli pesanti come l'editor **Tiptap** fuori dal bundle principale (`index.js`), abbiamo ridotto drasticamente la quantità di JavaScript che il browser deve scaricare, decodificare ed eseguire all'avvio.
3.  **Gestione degli Stati di Caricamento:** Introdotto un nuovo componente `Loader` con animazioni fluide (Framer Motion) per gestire le transizioni tra i chunk caricati pigramente, mantenendo un feedback visivo coerente con il design system.

#### Risultati del Build:
- **Prima:** Un unico grande file JS contenente tutto il codice del sito (~1MB+).
- **Dopo:**
    - `index.js` (~560kB): Contiene solo React, il layout principale e la homepage.
    - `ArticleEditor.js` (~470kB): Contiene Tiptap e le sue estensioni, caricato solo nell'admin.
    - Altri chunk minori (~5-20kB) per ogni singola pagina.

## 🔍 Dettagli Tecnici

### React.lazy & Suspense
Abbiamo utilizzato le API native di React 19 per definire le rotte del router in modo dinamico:
```tsx
const ArticleEditor = React.lazy(() => import('./pages/admin/ArticleEditor'));
// ...
<React.Suspense fallback={<Loader />}>
  <RouterProvider router={router} />
</React.Suspense>
```
Questo approccio garantisce che Vite generi automaticamente dei chunk separati durante il build, ottimizzando la distribuzione delle risorse.
