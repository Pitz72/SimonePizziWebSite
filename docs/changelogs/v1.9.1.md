## 🚀 Miglioramenti Performance

### [P2-05] Ottimizzazione Bundle & Code Splitting — COMPLETATO ✅

Per migliorare l'esperienza utente "Premium" e ridurre il peso del primo caricamento (First Contentful Paint), abbiamo implementato una strategia di **Code Splitting** aggressiva.

#### Cambiamenti Architetturali:
1.  **Lazy Loading delle Rotte:** Tutte le pagine dell'area **Admin** e le sottopagine pubbliche (Contatti, Progetti, Articoli singoli) vengono ora caricate solo quando effettivamente richieste dall'utente.
2.  **Riduzione Bundle Iniziale:** Spostando moduli pesanti come l'editor **Tiptap** fuori dal bundle principale (`index.js`), abbiamo ridotto drasticamente la quantità di JavaScript che il browser deve scaricare, decodificare ed eseguire all'avvio.

---

### [P2-06] Refactor Data Loaders (React Router 7) — COMPLETATO ✅

Abbiamo completato la migrazione all'architettura **"Render-as-you-Fetch"** utilizzando i loader nativi di React Router 7.

#### Cambiamenti Architetturali:
1.  **Eliminazione useEffect:** Rimosso il pattern di fetching dei dati all'interno del ciclo di vita del componente (`useEffect`). I dati vengono ora pre-caricati dal router durante la transizione di rotta.
2.  **Zero Layout Shift:** Grazie ai loader, i componenti admin non mostrano più stati di caricamento "a freddo" o salti di layout durante l'inizializzazione dei dati.
3.  **Parallelismo Network:** Utilizzato `Promise.all` in `src/loaders.ts` per caricare simultaneamente articoli, categorie e tag, riducendo drasticamente la latenza percepita.
4.  **Sincronizzazione URL:** Liste e filtri (es. in `ProjectsList`) sono ora sincronizzati con i `searchParams`, rendendo la navigazione admin pienamente gestibile tramite cronologia del browser.
