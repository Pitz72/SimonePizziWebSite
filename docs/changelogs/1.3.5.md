# 1.3.5 - Iniezione del Database nel Frontend

**Data:** 28 Febbraio 2026

Il momento in cui la Single Page Application (SPA) si stacca dal cordone ombelicale dei dati statici (TypeScript Dummy Data) e impara a respirare interrogando da sola le API in real-time.

## ⚙️ Dettagli dell'Intervento (Fetching Live)
1. **L'intermediario React (`useFetchArticles.ts`)**: Creato un possente custom-hook che chiama l'endpoint `articles.php` tramite il pre-testato modulo `api.ts`.
   *   Mappa al volo il DTO relazionale in uscita dal DB SQLite (id, title, excerpt...) dentro le rigide interfacce TypeScript dell'app (il vecchio type `PortfolioItem`), fungendo da traduttore simultaneo, garantendo così una retro-compatibilità assoluta coi componenti React visivi pre-esistenti senza romperne nemmeno uno.
2. **Lo sblocco della Home Page (`PortfolioGrid.tsx`)**: Rimosso totalmente l'import a `portfolioData.ts`. La griglia "Ultimi Lavori" ora chiama l'hook, filtra gli articoli con flag *Vetrina*, li ordina per tempo/rango e accende loader-spinners ad alta fluidità visiva (`Tailwind animate-spin`) se il payload delle API dovesse ritardare più di 100ms.
3. **Pagine Categoria Espugnate (`PortfolioShowcase.tsx`)**: Così come la Home, anche i layout interni "Videogiochi/Focus", "Libri" o "Software" non dipendono più dalle Props in App.tsx. I componenti estraggono l'URL della macro-pagina in cui si trovano, contattano l'API e renderizzano dinamicamente in base a ciò che il proprietario pubblicherà tramite il "Mini-CMS Editor" dello Step 4.
4. **Pulizia Totale in `App.tsx`**: Tranciate via tutte le dipendenze statiche nel Router: il passaggio del testimone è definitivo. Se la `build strict` fallisce significa che rimangono orfani pezzi sparsi di dati statici TS; ora `Exit code: 0`.
