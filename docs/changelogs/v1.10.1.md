# Changelog v1.10.1 — Database Schema Alignment & Backend Fixes
**Data:** 24 Aprile 2026

Questa versione risolve i problemi critici di backend emersi dopo il refactoring della pagina contatti, allineando lo schema del database MySQL online alle specifiche del codice.

## Novità e Miglioramenti

### 1. Database Alignment
- Risolto errore 500 nel form contatti tramite l'aggiunta delle colonne mancanti (`subject`, `read_at`) nella tabella `messages`.
- Allineata la tabella `subscribers` con i campi necessari per la gestione della newsletter (`name`, `confirm_token`, `unsubscribe_token`, `confirmed_at`).
- Creata la tabella `newsletter_sends` per supportare lo storico degli invii, precedentemente mancante nel database di produzione.

### 2. Robustezza Backend
- Aggiornata la gestione degli errori in `messages.php`, `subscribers.php` e `newsletter_send.php`.
- Implementata la cattura di `Throwable` (anziché solo `Exception`) per evitare crash silenziosi e fornire messaggi diagnostici chiari in caso di fallimento.

### 3. Sicurezza e Pulizia
- Rimosso lo script di emergenza `emergency_db_fix.php` dopo l'allineamento riuscito del database.
- Ottimizzata la connessione PDO per garantire la corretta gestione dei charset `utf8mb4`.

## Dettagli Tecnici
- **SQL**: Eseguiti comandi `ALTER TABLE` per l'aggiornamento dello schema senza perdita di dati.
- **PHP**: Migliorata la verbosità degli errori per facilitare futuri debug.
