## ✨ Nuove Feature

### Sistema Newsletter Completo [P1-03]

#### Backend PHP

**`public/api/subscribers.php`** — API iscrizioni completa:
- `POST` pubblico — nuova iscrizione con double opt-in: genera `confirm_token` e `unsubscribe_token` (64 char casuali), salva con `status=pending`, invia email HTML di conferma.
- `GET ?action=confirm&token=XXX` — conferma iscrizione: imposta `status=confirmed`, `confirmed_at=NOW()`, cancella token.
- `GET ?action=unsubscribe&token=XXX` — disiscrizione via token nel footer email.
- `GET` (admin) — lista iscritti con stats (totale / confermati / in attesa / disiscritti).
- `DELETE ?id=N` (admin) — elimina iscritto.

**`public/api/newsletter_send.php`** — Invio newsletter:
- `POST` (admin) — invia newsletter HTML a tutti gli iscritti `confirmed`. Ogni email include link di disiscrizione personalizzato. Salva l'invio in `newsletter_sends` con conteggio destinatari.
- `GET` (admin) — storico newsletter inviate.

**`public/api/stats.php`** — Sbloccato il TODO placeholder: ora legge il count reale degli iscritti confermati dalla tabella `subscribers`.

#### Database — Migration v1.7.4
Script `scripts/migrate_newsletter.php` (da copiare in `public/api/`, aprire da browser, poi eliminare):
- `ALTER TABLE subscribers`: aggiunte colonne `name`, `confirm_token`, `confirmed_at`, `unsubscribe_token`.
- `CREATE TABLE newsletter_sends` (`id`, `subject`, `body`, `sent_at`, `recipient_count`).

#### Frontend Pubblico

**`src/components/NewsletterSignup.tsx`** — Componente form iscrizione:
- Modalità `compact` (email + pulsante in riga) per Footer.
- Modalità standard (card con nome + email + pulsante) per singolo articolo.
- Gestione stati: idle / loading / success / already / error.
- Feedback visivo chiaro: "Controlla la tua email per confermare l'iscrizione."

**`src/pages/NewsletterConfirm.tsx`** — Pagina `/newsletter/confermato`:
- Legge il token dall'URL, chiama `subscribers.php?action=confirm`, mostra successo o errore.

**`src/pages/NewsletterUnsubscribe.tsx`** — Pagina `/newsletter/disiscritto`:
- Legge il token dall'URL, chiama `subscribers.php?action=unsubscribe`, mostra conferma disiscrizione.

**Embed widget:**
- `Footer.tsx` — widget newsletter compatto sempre visibile sopra la sezione PayPal.
- `SingleArticle.tsx` — card newsletter standard sotto il box CTA a fine articolo.

#### Admin

**`src/pages/admin/NewsletterAdmin.tsx`** — Pannello newsletter con due tab:
- **Iscritti:** contatori (totale/confermati/in attesa/disiscritti), tabella con badge stato, eliminazione singolo iscritto, export CSV.
- **Invia Newsletter:** campo oggetto, textarea corpo (supporta testo, HTML, Markdown), pulsante con contatore iscritti confermati, storico invii.

**`src/pages/admin/AdminLayout.tsx`** — Aggiunta voce "Newsletter" con icona `Mail` nella sidebar.

**`src/App.tsx`** — Aggiunte route:
- `/newsletter/confermato` → `NewsletterConfirm`
- `/newsletter/disiscritto` → `NewsletterUnsubscribe`
- `/admin/newsletter` → `NewsletterAdmin`

**`src/api.ts`** — Aggiunti 6 metodi: `newsletterSubscribe`, `newsletterConfirm`, `newsletterUnsubscribe`, `getSubscribers`, `deleteSubscriber`, `sendNewsletter`, `getNewsletterHistory`.

## 🚀 Deploy — Istruzioni

1. Caricare `dist/` via FTP (build pulita, senza file sensibili).
2. Copiare `scripts/migrate_newsletter.php` in `public/api/migrate_newsletter.php` sul server.
3. Aprire da browser: `https://simonepizzi.runtimeradio.it/api/migrate_newsletter.php`.
4. Verificare output: tutti i log devono essere `[OK]` o `[SKIP]`.
5. **Eliminare immediatamente** `migrate_newsletter.php` dal server.
