# Changelog v1.7.10 (2026-04-20)

## Sincronizzazione Categorie Dinamiche

In questa versione è stata completata l'integrazione del sistema di categorie dinamiche introdotto nelle versioni precedenti, estendendolo a tutte le parti del sito che erano rimaste legate a liste statiche hardcoded.

### Novità e Miglioramenti

- **Dashboard - Editor Progetti**: Il selettore della categoria ora carica dinamicamente le voci dal database invece di usare una lista fissa.
- **Dashboard - Editor Articoli**: Sostituita la lista categorie statica con il caricamento dinamico dall'API per garantire coerenza tra tutti i contenuti.
- **Dashboard - Elenco Progetti**: 
    - L'elenco dei progetti nella dashboard ora riconosce e visualizza correttamente i titoli delle nuove categorie.
    - Risolto un bug critico per cui i progetti inseriti in categorie appena create scomparivano dalla lista admin.
    - Il raggruppamento delle sezioni segue ora l'ordinamento (`sort_order`) definito nella dashboard.
- **Frontend - Tutti i Progetti**: 
    - La pagina pubblica è stata migrata all'utilizzo dell'hook `useCategories`.
    - Le sezioni dei progetti vengono generate automaticamente in base alle categorie attive nel DB.
- **Refactoring & Coerenza**: Centralizzato l'uso dell'hook `useCategories` per una gestione più robusta dei fallback e del caricamento.

### Correzioni Tecniche
- Rimosse costanti duplicate e hardcoded (`CATEGORIES`, `CATEGORY_OPTIONS`, `CATEGORY_LABELS`) in favore di un'unica fonte di verità (il DB).
- Risolto errore di build TypeScript relativo a variabili inutilizzate nel componente `ProjectsList`.
- Ottimizzato il caricamento parallelo di categorie e dati negli editor per ridurre i tempi di attesa.
