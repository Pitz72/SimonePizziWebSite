# Changelog v1.7.12

**Data:** 20 Aprile 2026

## 🚀 Nuove Funzionalità e Architettura Core
- **Gestore Tag Relazionale [P2-04]**: Il sistema dei tag è stato staccato dalle singole stringhe separate da virgole collocate colonna per colonna, passando ad una stabile struttura scalare tramite tabella dedicata (`tags`) e ponte logico molti a molti (`article_tags`).
- **Dashboard Tag (Mini-CMS)**: Integrata l'interfaccia interattiva nel menu per correggere in blocco e revisionare i singoli tag evitando duplicazioni mal digitate nel flusso di lavoro redazionale. In più monitoraggio effettivo dell'utilizzo di specifici tag inseriti in circolo.
- **Supporto multi-token nel Rich Editor**: In `ArticleEditor`, un input ibrido accetta inserimenti da input tastiera segnando a schermo come "Badge/Capsula" i tag da impilare usando anche datalist `autocomplete` coi tag disponibili nel db.
- **Paginazione Nativa Backend-Driven [P3-01]**: Invece di iniettare 100 articoli alla navigazione, `api/articles.php` conta ed emette pacchetti predefiniti su `OFFSET` ed il client di `ArticleArchive.tsx` si occupa di accodarli in memory array tenendo a mente quando si tocca `hasMore: false`. Limitato `PortfolioGrid` a blocchi espliciti da 7 in chiamata per fluidità.

## 🛠️ Deploy Info
- Script transitorio `public/api/migrate_tags.php` creato specificamente per migrare in automatico le virgole storiche su nuove chiavi ID DB. Da mandare in sleep dopo uso post-FTP deploy in remoto.
- Retrocompatibilità su Client Front: `GROUP_CONCAT` integrato nel GET per non corrodere il parser TS legacy del view mode.
