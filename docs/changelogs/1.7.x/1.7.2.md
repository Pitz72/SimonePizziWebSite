# Changelog v1.7.2 (06-04-2026)

## Bugfix Editor Articoli e Caricamento Immagini

Questa versione risolve un bug critico di integrità dei dati nell'editor degli articoli e introduce una nuova funzionalità per il caricamento multimediale contestuale.

---

## 🛠️ Bugfix: Integrità Post e Protezione Payload

### Il Problema (Articoli "Senza Titolo")
È stato identificato un bug che portava alla creazione di articoli "Senza Titolo" e alla perdita del contenuto originale durante la modifica. 
**Causa:** L'incollaggio diretto di immagini (Base64) nel corpo del testo generava payload JSON troppo grandi per i limiti del server (PHP `post_max_size` o MySQL `max_allowed_packet`). Quando il server rifiutava la richiesta, il backend riceveva dati nulli e applicava i valori di default ("Senza Titolo").

### Fix Applicato
- **Backend (`articles.php`):** Implementata una validazione rigorosa del payload JSON. Se la decodifica fallisce o il payload è vuoto a causa di limiti superati, il server restituisce ora un errore **401/400** invece di sovrascrivere l'articolo con dati vuoti.
- **Frontend (`RichTextEditor.tsx`):** Implementato un filtro sull'evento *Paste*. Se l'utente tenta di incollare immagini Base64, l'azione viene bloccata e viene mostrato un avviso che suggerisce l'uso del nuovo strumento di caricamento.

---

## ✨ Nuova Feature: Caricamento Immagini Contestuale

Aggiunta la possibilità di caricare immagini direttamente dall'interfaccia dell'editor senza passare preventivamente dalla Media Gallery.

- **Nuovo Pulsante Toolbar:** Aggiunta l'icona <ImageIcon size={16}/> per l'upload rapido.
- **Logica di Upload:** Integrazione con `api/upload.php`. L'immagine selezionata viene caricata, convertita in WebP e inserita nel testo nel punto esatto del cursore.
- **Styling Automatico:** Le immagini inserite tramite l'editor ricevono automaticamente classi di stile per essere responsive e visualmente coerenti con il design del sito (`max-w-full`, `rounded-xl`, `shadow-lg`).

## 🧹 Ottimizzazione Build e Sicurezza

- **`clean-dist.js`**: Aggiornato per garantire l'esclusione di file sensibili (`config.php`, `.data/`, `uploads/`) durante la generazione della cartella `dist/`, facilitando il deploy via FTP in ambienti con database MySQL delocalizzato.
- **Versione**: Progetto portato ufficialmente alla versione **1.7.2**.
