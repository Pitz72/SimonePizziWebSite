# Roadmap Evolutiva — Simone Pizzi Website
**Data:** 25 Aprile 2026  
**Stato:** Pianificazione Futura Post-v1.11.1

Questo documento delinea le aree di miglioramento necessarie per trasformare il progetto da un portfolio consolidato a una piattaforma di livello enterprise completamente conforme e scalabile.

---

## 🏗 Infrastruttura & Architettura
1. **Automazione Prerendering**: Implementare trigger automatici (via PHP/MySQL hooks) per rigenerare le pagine statiche al salvataggio di un articolo, eliminando il processo manuale.
2. **Framework Backend**: Valutare la migrazione delle API verso un framework robusto (es. Laravel o Slim) per una gestione più sicura delle rotte, del middleware e dei test.
3. **PWA & Offline Support**: Trasformare il sito in una Progressive Web App per consentire la consultazione dei progetti anche in assenza di rete (modalità fiera/presentazione).
4. **WebSocket**: Introdurre socket per notifiche push istantanee lato admin (nuovi messaggi, nuove iscrizioni) e aggiornamenti live dei contenuti.

## 📈 SEO & Analytics
5. **Risoluzione Bug Robots.txt**: Debug profondo dei conflitti Apache/.htaccess per garantire la stabilità del crawling.
6. **Integrazione API Esterne**: Collegare la dashboard admin alle API di Google Search Console per monitorare query e posizionamento senza uscire dal CMS.
7. **Sitemap Dinamica**: Passare da una sitemap fisica generata a una sitemap dinamica servita in tempo reale per riflettere istantaneamente le nuove pubblicazioni.

## 🔐 Sicurezza & Gestione
8. **Autenticazione a Due Fattori (2FA)**: Aggiungere un layer di sicurezza (es. Google Authenticator) per l'accesso all'area amministrativa.
9. **Supporto Multi-utente**: Implementare la gestione di ruoli e permessi per consentire l'accesso a redattori o collaboratori con privilegi limitati.
10. **Test Automatizzati**: Introduzione di unit test (PHPUnit/Jest) e test d'integrazione (Cypress/Playwright) per prevenire regressioni durante le future evoluzioni.

## 🎨 User Experience & Design
11. **Localizzazione (i18n)**: Implementare il supporto multi-lingua, dando priorità alla versione inglese per una portata internazionale.
12. **Inclusività Visiva (Light Mode)**: Progettare un tema chiaro alternativo per rispondere a diverse esigenze di accessibilità e preferenze ergonomiche.
13. **Ottimizzazione Media Dinamica**: Implementare il servizio delle immagini con risoluzioni variabili (srcset dinamico) basato sulle dimensioni dello schermo del client.

## ⚖️ Compliance & Community
14. **Conformità GDPR**: Integrazione di un Cookie Banner a norma e creazione di pagine dedicate a Privacy e Cookie Policy.
15. **Interattività & Social**: Valutare l'inserimento di un sistema di commenti moderato o di reazioni per favorire il coinvolgimento dell'utenza.
