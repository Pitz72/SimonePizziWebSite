## 🗳️ Piano Dettagliato — Reazioni agli Articoli (Punto 14)

### Analisi di Fattibilità (6 righe)

Il sistema di reazioni è fattibile in una singola sessione di lavoro: richiede una nuova colonna JSON `reactions` nella tabella `articles` (o una tabella separata `article_reactions` per evitare lock), un endpoint PHP `POST /api/reactions.php` senza autenticazione, e un componente React `ReactionBar.tsx` client-side. Per prevenire voti multipli senza richiedere un account si usa una chiave composita `ip + article_id` su una tabella dedicata oppure un cookie/localStorage lato client (approccio più leggero e privacy-friendly). Le emoji candidate sono 5: 👍 Utile · ❤️ Bello · 🔥 Interessante · 🤔 Fa pensare · 🎮 Game-related (coerente col brand). Il componente si aggancia al SEO Engine senza modifiche: non richiede SSR perché le reazioni sono interazione pura client-side, visibili solo dopo l'idratazione React.

### Architettura Tecnica

**Database — nuova tabella:**

```sql
CREATE TABLE article_reactions (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    article_id  INT NOT NULL,
    emoji       ENUM('👍','❤️','🔥','🤔','🎮') NOT NULL,
    voter_hash  VARCHAR(64) NOT NULL,   -- SHA256(ip + user_agent) anonimo
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY unique_vote (article_id, voter_hash, emoji),
    INDEX idx_article (article_id)
);
```

**Endpoint PHP** — `public/api/reactions.php`:
- `GET ?article_id=X` → restituisce conteggi per ogni emoji
- `POST {article_id, emoji}` → aggiunge/rimuove reazione (toggle), protetto da rate limiting IP

**Componente React** — `src/components/ReactionBar.tsx`:
- Mostra 5 bottoni emoji con contatore
- Stato locale con `useState` per risposta immediata (optimistic update)
- Chiamate a `api.getReactions(id)` nel loader di `SingleArticle` (già presente)
- Cookie `reacted_${articleId}` per persistenza locale senza account
- Animazione micro `scale(1.3)` al click con Framer Motion

**Posizione nel layout:**

```text
[Contenuto articolo]
────────────────────
[ReactionBar]         ← tra contenuto e CTA
[CTA PayPal/Condividi]
[NewsletterSignup]
```

### Punti di Attenzione

- **Privacy:** `voter_hash` è SHA256 anonimo, non identifica l'utente, conforme GDPR
- **Spam:** Rate limiting a livello PHP (max 10 reazioni/minuto per IP) + `UNIQUE KEY` su DB previene duplicati
- **Mobile:** Bottoni min 44px touch target, layout flex-wrap per schermi piccoli
